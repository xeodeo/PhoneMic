# PhoneMic v1.4

## Correcciones de bugs

### EXE no iniciaba el daemon de ADB en instalación nueva
El ejecutable compilado fallaba al conectar vía USB si el daemon de ADB no había sido iniciado previamente de forma manual. El problema era que `adb devices` se ejecutaba con un timeout de 3 segundos que podía expirar mientras ADB intentaba arrancar el daemon internamente. Ahora se llama `adb start-server` explícitamente antes de listar dispositivos — este comando bloquea hasta que el daemon esté listo y es un no-op si ya estaba corriendo.

### EXE no abría en instalación limpia (Python 3.14 / shiboken6)
El ejecutable compilado con Python 3.14 no mostraba ninguna ventana ni error. La causa raíz era una incompatibilidad fatal entre shiboken6 y Python 3.14 (`libshiboken/signature: could not initialize part 2`). La solución fue cambiar la compilación a Python 3.12 donde PySide6 y shiboken6 funcionan correctamente. Se agregó además manejo de errores con MessageBox para capturar fallos al cargar Qt antes de que la ventana principal esté disponible.

### Error de carga de DLL al iniciar el EXE (`DLL load failed importing QtWidgets`)
PyInstaller no incluía automáticamente las DLLs de PySide6 y shiboken6 en el ejecutable empaquetado. Se agregaron `--collect-binaries PySide6` y `--collect-binaries shiboken6` a los parámetros de compilación, y se registran los directorios de DLL con `os.add_dll_directory()` antes de importar Qt.

### Tags de versión con V mayúscula no se detectaban como actualización
La comparación de versiones en Windows y Android usaba `.lstrip("v")` que no eliminaba la V mayúscula. Los tags de GitHub con formato `V1.X` no se reconocían como nueva versión. Corregido en `phonemic_client.py`, `phonemic/ui/app.py` y `MainActivity.kt`.

### About dialog de Android mostraba versión hardcodeada
El diálogo "Acerca de" en Android tenía la versión escrita directamente como texto en lugar de usar la constante `CURRENT_VERSION`. Ahora usa `$CURRENT_VERSION` para mantenerse sincronizado automáticamente.
